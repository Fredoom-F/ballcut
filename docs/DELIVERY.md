# 剪球本地版交付说明

更新日期：2026-06-19  
本地服务版本：0.4.4

## 最快启动

1. 双击 `setup-jianqiu.cmd`，完成 Node.js、Python、OpenCV 和脚本检查。
2. 双击 `start-jianqiu.cmd`。
3. 打开 `http://127.0.0.1:4173/`。
4. 上传视频，确认运动类型和拍摄机位，再开始本地分析。

后台服务只监听 `127.0.0.1`。关闭启动窗口不会停止服务；需要停止时双击
`stop-jianqiu.cmd`。

## 建议使用流程

1. 查看素材预检，确认分辨率、时长、文件大小和预计分析耗时。
2. 选择运动类型；不确定拍摄角度时保留“自动”。
3. 夜场、偏色或特殊颜色球可先点击画面校准球颜色。
4. 完成分析后，先复核击球候选，再确认或恢复等待片段。
5. 查看候选命中率、置信度分档和导出准备度。
6. 按需要调整精彩阈值、集锦时长、输出比例、特效和声音。
7. 下载剪辑项目、标注数据、CSV、EDL、报告或带水印 WebM 预览。

## 数据边界

- 原视频不会上传到外部服务器。
- 上传接口只在本机回环地址工作，并限制来源、类型、容量和并发。
- OpenCV 临时视频在分析完成、失败或取消后删除。
- 分析缓存、人工复核和训练历史保存在当前浏览器本机存储中。
- JSON 标注数据不包含视频本体。
- 清除本机分析数据不会删除原视频。

## 识别结果应如何理解

- “球轨迹”来自颜色、运动区域、轮廓形状和连续位置约束。
- “疑似击球”来自连续轨迹的方向、速度和加速度变化，并结合镜头稳定性与活动区域代理。
- “等待/捡球”来自持续低运动量、没有可靠球轨迹和没有击球事件的时间段。
- “精彩瞬间”来自击球候选前后片段，再融合置信度、速度、方向变化、连续击球和人工确认排序。
- “人体活动区域代理”不是人体检测框，也不是姿态模型。
- 当前没有运行人体姿态或球拍检测模型，页面会明确显示为未启用。

## 人工复核闭环

- 击球候选可确认、忽略、收藏、重定位、分类和填写备注。
- 等待片段可确认删除或恢复保留。
- 模型候选命中率只统计已确认和已忽略的自动候选，不计手动添加事件。
- 高、中、低置信度分档命中率用于判断当前机位和阈值是否可靠。
- 标注 JSON 可用于后续模型训练、离线评估或教练复核。

## 跨电脑继续修改

```powershell
git clone https://github.com/Fredoom-F/ballcut.git
cd ballcut
.\setup-jianqiu.cmd
.\start-jianqiu.cmd
```

浏览器本机缓存不会随 Git 同步。需要继续同一个视频项目时：

1. 在旧电脑下载剪辑项目 JSON。
2. 将原视频和项目 JSON 一起带到新电脑。
3. 在新电脑先选择原视频，再导入剪辑项目。
4. 系统会校验视频时长、文件大小、首尾快速 SHA-256 指纹和项目数据范围。

需要迁移常用编辑设置和训练历史时，可在旧电脑下载“本机设置备份”，再在新电脑恢复。该备份不包含视频、分析缓存或当前剪辑项目。

## 当前限制

- 识别仍依赖 OpenCV 几何与颜色特征，不是裁判级判断。
- 球严重模糊、过小、与背景同色或长时间遮挡时，轨迹会中断。
- 手持快速移动、切镜和强烈光照变化会降低准确率。
- 浏览器当前导出 WebM；高清 MP4 仍需要后续接入 FFmpeg。
- 暂无云端账号、多人项目、支付和跨设备自动同步。

# 剪球

剪球是一个面向球类运动爱好者的本地运动剪辑 MVP。当前版本使用 Python、OpenCV 和 NumPy 对真实视频帧进行分析，可以在浏览器里完成视频载入、球轨迹追踪、疑似击球检测、冗余片段建议、击球特效预览、精彩瞬间时间线、带水印预览导出和剪辑决策 JSON 下载。

当前实现是本地最小价值单元：视频不会上传到外部服务器，页面和处理逻辑都运行在 `127.0.0.1`。

## 环境要求

- Node.js 18 或更高版本
- Python 3.10 或更高版本
- Python 包：`opencv-python`、`numpy`

安装 Python 依赖：

```powershell
python -m pip install opencv-python numpy
```

Windows 用户可直接双击 `setup-jianqiu.cmd` 完成依赖检查与安装，然后双击 `start-jianqiu.cmd` 启动。

## 运行方式

进入应用目录：

```powershell
cd app
npm start
```

然后打开：

```text
http://127.0.0.1:4173/
```

Windows 用户也可以直接双击根目录的 `start-jianqiu.cmd`。服务会在本机后台独立运行，关闭启动窗口不会中断。

需要停止后台服务时，可运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\stop-jianqiu.ps1
```

## 当前功能

- 本地视频上传与预览
- 生成演示视频，方便无素材时测试流程
- 运动类型选择：网球、羽毛球、乒乓球、篮球、足球、高尔夫
- 剪辑目标选择：综合剪辑、删除捡球等待、精彩瞬间、击球特效
- 从真实视频帧生成球轨迹、疑似击球点、冗余片段和精彩片段
- 上传与逐帧分析显示实时进度、处理帧率和预计剩余时间
- 支持取消分析并自动清理 OpenCV 子进程和临时视频
- 点击画面校准球颜色，适配夜场、偏色和不同训练球
- 保守、标准、灵敏三档击球识别灵敏度
- 播放时跳过冗余片段
- 时间线显示当前播放指针，点击任意位置可直接跳转
- 上一击/下一击快速导航和编辑键盘操作
- 确认、忽略、收藏和手动添加击球
- 单独恢复误删的等待片段
- 分析结果及人工审阅状态保存在浏览器本地缓存
- 精彩分排序，精彩模式只导出高分或收藏片段
- 训练复盘、拍摄质量诊断和可分享 HTML 报告
- 最近训练趋势与可选历史基线对比，按视频时长归一化每分钟有效击球
- 对视频时长和机位稳定性差异给出比较提示，避免把拍摄条件变化误判为训练进步
- Canvas 击球特效预览
- 导出带水印 WebM 预览视频
- 下载剪辑决策 JSON

## 当前识别判据

- 球候选：颜色范围与帧间运动区域相交，再按面积、圆度、边缘距离和轨迹连续性评分。
- 球色校准：用户点击球后采样局部中位 RGB，转换成专用 HSV 范围；缓存键会包含校准颜色。
- 球轨迹：候选点必须与上一位置或预测位置保持合理距离；短时丢失后会终止当前轨迹，避免跨场景误连。只向前端展示镜头稳定期间的轨迹。
- 镜头稳定：使用相邻帧全局位移和画面运动面积判断手持移动；事件前后约 1 秒内镜头不稳定时拒绝击球候选。
- 球拍类击球：连续轨迹中球速达到最低门槛、球位于合理场地区域、靠近人体尺度运动区域，同时前后运动方向出现显著反向变化。发球时允许球位于运动者上方。
- 其他运动关键动作：允许使用显著方向变化，或伴随明显速度倍率变化的加速事件。
- 冗余片段：持续低运动量、没有移动球轨迹、也没有击球事件，并超过用户选择的时长阈值。
- 精彩片段：置信度不低于 0.58 的疑似击球事件前 2.2 秒至后 3 秒。
- 精彩分：融合事件置信度、方向变化、画面球速、轨迹连续性、连续击球密度和用户确认/收藏。

事件结果会保存方向变化角度、前后速度、加速度和轨迹连续度。没有形成可靠轨迹时，系统返回明确警告，不会生成模拟击球。

```powershell

```

测试会自动生成已知击球时间和等待区间的合成网球视频，并验证：

- 4 个击球全部命中
- 平均击球时间误差
- 轨迹覆盖率
- 等待区间识别
- 无球视频不得产生击球
- 本地分析接口可用
- 临时视频处理后被删除
- 前后端脚本语法
- 时间线播放指针与点击跳转代码契约
- 进度增长、ETA 上报、取消任务和临时文件清理
- 自定义球色在非默认颜色测试球上恢复全部已知击球

## 已知限制

- 当前是基于运动、颜色和轨迹几何的 OpenCV 版本，还没有加入人体姿态模型和球拍检测模型。
- 球太小、严重模糊、与背景同色、镜头快速移动或光线变化剧烈时，轨迹可能中断。
- 选择正确运动类型很重要，因为不同球类使用不同颜色和尺寸范围。
- “疑似击球”是轨迹变化判断，不等同于自动计分或裁判级识别。

## 真实视频本地回归

使用一段约 115 秒、3840×2160、30fps 的夜间网球手机视频进行过完整本地接口测试：

- 识别 9 个经前后帧人工核验的发球/击球候选
- 过滤空球场、捡球动作和手持镜头移动造成的误报
- 建议删除约 0～12.5 秒、42.1～50.4 秒和 87.8 秒之后的无效内容
- 从浏览器同款 `/api/analyze` 接口上传分析成功
- 分析完成后临时视频文件数量恢复为 0
- 异步任务采集 21 个进度样本，ETA 从约 113 秒逐步收敛至 0
- 使用 `grab/retrieve` 跳过非采样帧后，同机分析耗时由约 100 秒降低至约 50 秒，9 个核验击球和删除区间保持不变

该结果用于回归当前拍摄角度，不代表所有网球视频都能达到相同准确率。

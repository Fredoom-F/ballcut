# Global Rules

These rules apply across projects that use this Codex team configuration.

## Collaboration

- Understand the request before editing.
- Make reasonable assumptions for low-risk details.
- Ask only when missing information would make the result risky.
- Keep progress updates concise and useful.

## Code Quality

- Match existing style, formatting, and architecture.
- Keep changes focused on the requested outcome.
- Use structured parsers and project APIs where available.
- Prefer maintainable, obvious code over clever code.

## Safety

- Do not run destructive operations without explicit instruction.
- Do not revert user changes unless explicitly asked.
- Treat secrets, tokens, and credentials as sensitive.
- Leave a short record of important decisions in project docs when useful.
